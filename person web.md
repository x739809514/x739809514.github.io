搭建个人品牌网站是一个非常棒的想法！为了确保网站既美观又易于维护，我们需要一个**前后台分离**的架构。

以下我为你制定的项目需求文档（PRD）和技术选型方案。

---

## 1. 项目需求文档 (PRD)

### 1.1 项目目标

构建一个响应式、高性能的个人门户网站，用于展示个人形象、作品集（Gallery）并发布技术/生活笔记（Markdown）。

### 1.2 功能模块架构

* **前台展示页 (Client Side)**
* **首页 (Home):** 包含头像、姓名、简介及社交媒体链接（GitHub, LinkedIn, X）。
* **作品集 (Gallery):** 以瀑布流或网格形式展示。
* 卡片包含：缩略图、标题、简短描述（Detail）。


* **笔记 (Notes):** 列表展示笔记标题与日期，点击进入详情页，支持 Markdown 渲染。


* **管理后台 (Admin Side)**
* **身份认证:** 登录页面（仅允许你自己进入）。
* **信息管理:** 修改个人基础信息、更新社交链接。
* **作品管理:** 上传图片、编辑标题与详情、删除作品。
* **笔记管理:** 一个简单的 Markdown 编辑器，支持发布、编辑和删除。



---

## 2. 技术选型建议

考虑到你希望有**后台管理**且支持 **Markdown**，我推荐目前最流行的两种方案：

### 方案 A：全栈现代开发（高自定义，推荐）

适合想完全掌控视觉效果，且具备一定开发基础的场景。

* **前端框架:** **Next.js** (React 生态，SEO 友好，速度极快)。
* **样式:** **Tailwind CSS** (快速实现美观的响应式布局)。
* **后端/数据库:** **Supabase** (开源的 Firebase 替代品，提供现成的数据库、图片存储和用户登录，无需写复杂的后端代码)。
* **Markdown 解析:** `react-markdown` 或 `MDX`。

### 方案 B：无头 CMS 方案（开发最快）

适合想快速上线，专注内容创作的场景。

* **前端:** **Astro** 或 **Next.js**。
* **内容管理 (CMS):** **Strapi** 或 **Contentful** (这些工具自带非常漂亮的后台界面，你只需要在前端调用它们的 API)。

---

## 3. 详细技术路线图 (基于方案 A)

| 维度 | 推荐工具 | 理由 |
| --- | --- | --- |
| **框架** | **Next.js (App Router)** | 目前 Web 开发的标准，自带路由和服务器组件优化。 |
| **数据存储** | **Supabase (PostgreSQL)** | 免费额度大，不仅能存数据，还能存 Gallery 的图片。 |
| **笔记处理** | **Gray-matter + Remark** | 将 Markdown 文件转换为 HTML 并在前端渲染。 |
| **部署** | **Vercel** | Next.js 的母公司，代码 Push 到 GitHub 即可自动部署。 |
| **UI 组件库** | **shadcn/ui** | 极其精美的预制组件，非常适合做 Gallery 和后台表格。 |

## 4. 核心数据结构设计 (Schema)

在数据库中，你至少需要这两张表：

### Gallery 表

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | UUID | 唯一标识 |
| title | String | 作品标题 |
| detail | Text | 详细描述 |
| image_url | String | 图片存储路径 |
| created_at | Timestamp | 创建时间 |

### Notes 表

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | UUID | 唯一标识 |
| title | String | 笔记标题 |
| content | Markdown/Text | 笔记正文内容 |
| category | String | 分类（可选） |

---

## 5. 开发建议与后续步骤

1. **第一步：** 先在本地搭建一个 Next.js 项目。
2. **第二步：** 注册 Supabase，创建一个 Project，并把 API Key 配置到你的代码里。
3. **第三步：** 先做“静态版”前端页面（把照片、链接先写死），确保样式是你喜欢的。
4. **第四步：** 引入 Markdown 渲染插件，实现“点击标题看文章”的功能。
